ZTMUX // ENGINEERING REPORT

// a Rust port of tmux — full server + client · seeded from tmux-rs · ported against tmux C
Home Port Report GitHub
Status: early port, building & running. ztmux is the tmux program itself reimplemented in Rust — the server, the client, the grid/screen model, the input parser, layouts, the command language, formats, and the terminal back end. Not a wrapper around the tmux binary and not control mode. The crate compiles and runs (ztmux -V, ztmux new-session); the "done right" work of shrinking the unsafe surface carried over from the transpile is in progress.

How the port is built

ztmux stands on two references, both vendored under vendor/ as plain, read-only committed copies (not submodules) so the repo is self-contained. The Rust crate at the repo root was seeded from the tmux-rs base, then taken over as our own living code and ported forward against the C source of truth.

referencewhatrole
vendor/tmuxupstream tmux C sources (pinned)source of truth — every ported module is diffed against its C counterpart here
vendor/tmux-rsrichardscollin/tmux-rs (ISC, pinned)head start — src/ was seeded from its src/; kept pristine so git diff shows our divergence
src/the ztmux portthe living code — the crate we own and evolve toward safe, idiomatic Rust

Per-function port coverage

The port report is generated by scripts/gen_port_report.py: it indexes every C function in vendor/tmux/**/*.c, maps each C file to its Rust counterpart in src/ (dash→underscore, with the tmux-rs trailing-underscore convention), and reports per-symbol and per-file coverage — line counts, call-site counts, and a call-% "under-wired" signal (a Rust port that exists but isn't called at the C-equivalent sites). Regenerate with:

python3 scripts/gen_port_report.py

"Done right"

  1. Start from a working skeleton — the tmux-rs base is alpha-quality and almost entirely unsafe, a faithful mechanical transpile. It gives us a running program to refactor rather than a blank page.
  2. Shrink the unsafe surface — replace raw-pointer intrusive lists, hand-rolled C idioms, and unsafe blocks with safe Rust equivalents where behavior allows.
  3. Verify against C at every step — the pinned vendor/tmux tree is ground truth; a module isn't "ported" until its behavior matches the C reference.
  4. Keep it green — cargo build and cargo clippy stay clean under the inherited lint policy as code comes over.

Build

Requires a C libevent and a terminfo database (ncurses), matching tmux itself. On macOS the build links Homebrew's libevent automatically (TMUX_RS_DISABLE_HOMEBREW_LIBS=1 to opt out). The binary is ztmux; it opens its own socket namespace under ztmux-<uid> so it never collides with a real tmux server.
