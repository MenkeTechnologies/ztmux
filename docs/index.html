Not a wrapper around the tmux binary, not control mode: the program itself.

In one line

ztmux is seeded from tmux-rs (a largely-unsafe Rust transpile of tmux) and ported forward against the upstream tmux C sources — both vendored, read-only, and pinned by SHA — with the goal of turning that working skeleton into safe, idiomatic Rust while keeping behavior verified against the C reference. Distinct from ztmux-core, which is a native tmux wire-protocol client engine; this repo is the multiplexer itself.
