# =====================================================================
# ztmux fake-fn allowlist — READ THE HEADER BEFORE ADDING A LINE
# =====================================================================
#
# Enforced by tests/ported_fn_names_match_c.rs, which runs in the CI
# `test` gate. That test flags every free `fn` in src/**.rs whose name
# does NOT exist as a function in the vendored tmux C sources
# (vendor/tmux/**/*.c + cmd-parse.y). ztmux is a PORT of tmux; a Rust
# function with no C counterpart is a "fake fn" that inflates apparent
# port progress. New ones FAIL THE BUILD.
#
# This file is the frozen snapshot of pre-existing, accepted exceptions.
# It is an AUDIT TRAIL, not a free pass: the goal is to SHRINK it, never
# grow it. Before adding a name, STOP — port the function against the C
# source, or inline it, instead. Adding a line needs a real reason.
#
# The current entries fall into these categories:
#
#   1. Macro-paired helpers (`foo_`) — C's variadic `foo(fmt, ...)` is
#      ported as a Rust `macro_rules! foo` (which carries the real C
#      name) plus a `foo_` helper fn the macro forwards to. The C name
#      IS present (as the macro); the `_` fn is its impl detail.
#      e.g. cfg_add_cause_, cmdq_print_, format_add_, control_write_.
#
#   2. C-fn-split variants (`foo_`, `foo__`, `foo___`) — one C function
#      (e.g. xstrdup) ported as several Rust overloads for different
#      argument types. These SHOULD be merged back toward the single
#      C-named function over time; kept here until then.
#      e.g. xstrdup_/__/___, vis_/__, utf8_strvis_, checkshell_.
#
#   3. libc / libevent shims — wrappers around C library functions that
#      are not part of tmux's own source (so they have no vendor/tmux
#      definition): evtimer_*, evbuffer_*, EVBUFFER_*, fopen, fnmatch,
#      basename, memcpy_, free_, __bswap_32, etc.
#
#   4. Rust glue — genuinely Rust-only helpers with no C analog:
#      _s, cstr_to_str, concat_array, assume_init_ref, format_nul_, etc.
#
# Regenerate after intentional churn:
#   cargo test --test ported_fn_names_match_c -- --nocapture 2>&1 \
#     | sed -n 's/^FAKE-FN \([A-Za-z0-9_]*\).*/\1/p' | sort -u
# (then re-add this header)
# =====================================================================
